## How it works

The `.dc.html` files are rendered by `support.js`, a small client-side runtime that resolves shared components (`SiteNav`, `SiteFooter`) and template bindings at load time. This requires the files to be served over HTTP, which is why the preview lives here rather than as files you open locally.

## Notes

- These are design prototypes, not production code. They are intended as a faithful visual reference for implementation.
- `assets/` holds product and brand imagery. `uploads/` holds source references and the design-system notes.
